# V-Assure Internal Validation Platform

> An internal tool built for Spotline Inc. to automate the validation and comparison of Veeva Vault test scripts — comparing client-provided template PDFs against V-Assure executed output PDFs with structured diff analysis, side-by-side visual review, and screenshot tracing.

---

## Overview

When validating Veeva Vault implementations, testers work with two PDF documents:

- **Template (Client Test Script)** — the pre-approved script provided by the client, containing Pre-Test Setup (PTS) instructions and numbered execution steps with expected results.
- **Report (V-Assure Output)** — the executed test script generated by V-Assure, containing actual results, screenshots, and runtime data.

Manually comparing these two documents is time-consuming and error-prone. This platform automates the comparison, surfaces structural and textual differences, and provides a rich review interface so validators can efficiently verify test execution and raise issues for developers.

---

## Features

### Analysis Tab
- Uploads both PDFs and sends them to a FastAPI backend for structured comparison
- Detects and categorises differences into:
  - **PTS / Setup Step mismatches** — procedure text differs between template and report
  - **Execution Step mismatches** — procedure mismatch, expected vs actual mismatch, or missing steps
  - **Dynamic data validation** — accepts runtime-generated values (emails, IDs) as valid matches
- **Word-level diff highlighting** on every mismatch — changed words are highlighted inline using a GitHub-style red/green word diff (via `DiffText.tsx`)
- Summary statistics: total issues, setup issues, execution issues

### Side-by-Side PDF Tab
- Renders both PDFs page-by-page in the browser using PDF.js at full resolution (3× scale + devicePixelRatio, PNG lossless)
- **Isolated scroll** per column — scrolling one side does not affect the other by default
- **Sync scroll toggle** — enable when both PDFs have the same page count to lock scrolling together
- **Text-layer diff highlights** — words that differ between the two documents are highlighted red (removed) or green (added) directly on the rendered PDF canvas
- **Zoom controls** from 50% to 350%
- **Fullscreen mode** via the native browser fullscreen API

### Screenshots Tab
- Scans every page of the executed PDF for screenshot labels matching:
  - `PTS Step #3 Screenshot #1` — Pre-Test Setup screenshots
  - `Step #16 Screenshot #1` — Execution step screenshots
- Renders each screenshot page at 3× quality
- Displays alongside the corresponding **Procedure** and **Expected Results** extracted directly from the same report PDF page
- PTS step text is shown with **line breaks preserved**, matching the indented bullet format in the original document
- Filter by All / PTS / Execution
- Steps with detected differences are marked with a **DIFF** badge

### Validation Notepad (Floating Widget)
- Persistent orange bubble in the bottom-right corner — click to expand
- **Ctrl+V anywhere** on the page pastes a screenshot directly into a new note
- Notes include: tag (Text Incorrect / Screenshot Incorrect / Step Incorrect / Other), step reference, and free-text comment
- Copy individual note (text + image via ClipboardItem API) or copy all notes at once
- Designed for validators to collect evidence during review, then paste into FreshDesk tickets

## Getting Started

### Prerequisites

- Python 3.10 or higher (3.11 recommended)
- Node.js 18 or higher
- npm 9+

### Backend Setup

```bash
cd backend

# Create and activate virtual environment
python -m venv venv

# Windows
venv\Scripts\activate
# macOS/Linux
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Start the API server
uvicorn app:app --reload --port 8000
```

The API will be available at `http://127.0.0.1:8000`.
Interactive docs at `http://127.0.0.1:8000/docs`.

### Frontend Setup

```bash
cd frontend

# Install dependencies
npm install

# Start the dev server
npm run dev
```

The app will be available at `http://localhost:8080`.

### Vite Proxy Configuration

The frontend proxies API calls through Vite's dev server to avoid cross-origin issues. Ensure your `vite.config.ts` includes:

```ts
server: {
  port: 8080,
  proxy: {
    '/compare': {
      target: 'http://127.0.0.1:8000',
      changeOrigin: true,
    }
  }
}
```

## How Comparison Works

### PTS (Pre-Test Setup) Steps

The backend extracts PTS steps from the template's header table using `pdfplumber.find_tables()` with `extract_words()` for accurate bbox-based cell reading. Steps are parsed by splitting on numbered item boundaries (`1.`, `2.`, `3.`). The comparison checks:

1. **Account steps** — validates that runtime email addresses have been filled in where the template requires usernames
2. **Procedure mismatch** — normalised text comparison with whitespace/case tolerance
3. **Missing steps** — steps present in template but absent in report

### Execution Steps

Execution steps are extracted from the table on pages 3+ of each PDF (`Step # | Procedure | Expected Results | Actual Results | Pass/Fail`). The comparator:

1. Normalises both procedure strings (lowercase, collapse whitespace)
2. Checks for **dynamic suffixes** — values that are runtime-generated (e.g. document names with appended IDs) are accepted as valid matches
3. Compares expected results against actual results
4. Flags `procedure_mismatch`, `expected_mismatch`, `expected_vs_actual_mismatch`, or `missing`

## Known Limitations

- The Clipboard API (`ClipboardItem` with image) for copying screenshots with images requires Chrome or Edge. Firefox copies text only.
- Screenshot step detection relies on the executed PDF containing labels in the exact format `PTS Step #N Screenshot #N` or `Step #N Screenshot #N`. Non-standard labels will not be detected.
- Side-by-side sync scroll is only meaningful when both PDFs have the same number of pages.
